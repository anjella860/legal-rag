> **작성일:** 2026년 6월
> **작성자:** 전보경
> **버전:** v1.0

## 1. 프로젝트 개요

### 1.1 개발 배경

일반 시민이 근로기준법·최저임금법 등 노동 관련 법령을 찾아볼 때, 법령 원문은 전문 용어가 많고 조문 구조가 복잡해 필요한 정보를 빠르게 파악하기 어렵다. 기존 법령 검색 서비스는 키워드 일치 방식에 의존하여 질문의 맥락을 이해하지 못하고, 관련 조문 전체를 반환하는 방식이라 사용자가 직접 내용을 해석해야 하는 부담이 있다.

본 프로젝트는 RAG(Retrieval-Augmented Generation) 기술을 활용하여 자연어 질문에 대해 관련 법령 조문을 검색하고, LLM이 이를 바탕으로 이해하기 쉬운 답변을 생성하는 법령 문서 검색 시스템을 구축한다.

### 1.2 개발 목적

- 노동법령 정보에 대한 시민 접근성 향상
- RAG 파이프라인 설계 및 구현 역량 확보 (LangChain, 벡터DB, 임베딩)
- FastAPI 기반 AI 백엔드 + Vue.js 프론트엔드 풀스택 포트폴리오 구성
- TAS 팀 프로젝트(CV 도메인)와 차별화되는 NLP 도메인 역량 증명

### 1.3 기대 효과

| 대상 | 기대 효과 |
|------|-----------|
| 일반 시민 | 연차·퇴직금·최저임금 등 노동 관련 궁금증을 자연어로 빠르게 해결 |
| 취업 준비생 | 근로계약·수습기간·해고 요건 등 권리 정보 손쉽게 확인 |
| 소규모 사업자 | 근로계약 작성·퇴직금 지급 기준 등 노무 관련 정보 신속하게 파악 |

## 2. 프로젝트 범위

### 2.1 구현 범위 (In-Scope)

- **법령 데이터 수집:** 국가법령정보센터 Open API를 통한 노동 관련 법령 원문 수집
- **데이터 전처리:** 조문(Article) 단위 청크 설계 + 메타데이터 태깅
- **벡터 임베딩 및 저장:** 임베딩 모델 적용 후 벡터 DB 저장
- **RAG 파이프라인:** 질문 임베딩 → 유사 조문 검색 → LLM 답변 생성
- **출처 표시:** 답변 근거가 된 법령명·조문번호 함께 반환
- **백엔드 API:** FastAPI 기반 REST API 서버
- **프론트엔드:** Vue.js 기반 검색 UI (질문 입력 → 답변 + 출처 표시)
- **회원가입 / 로그인:** JWT 기반 인증, 비회원/회원 권한 분리
- **질의응답 히스토리 저장:** 회원 전용 히스토리 DB 저장 및 조회
- **컨테이너화:** Docker Compose 기반 배포 환경 구성

### 2.2 제외 범위 (Out-of-Scope)

- 법률 상담 또는 법적 효력이 있는 해석 제공 (면책 문구 포함)
- 실시간 법령 개정 자동 반영
- 소셜 로그인
- 노동법령 외 타 도메인 법령 (민법, 형법 등)

## 4. 기술 스택

### 4.1 전체 스택 구성

| 분류 | 기술 | 선택 이유 |
|------|------|-----------|
| **AI / RAG** | LangChain | RAG 파이프라인 표준 프레임워크, 체인 구성 유연성 |
| **임베딩 모델** | jhgan/ko-sroberta-multitask | 한국어 특화 Sentence Transformer |
| **벡터 DB** | ChromaDB | 로컬 실행 가능, 개인 프로젝트 규모에 적합 |
| **LLM** | OpenAI GPT-4o-mini | 비용 효율적, 한국어 답변 품질 우수 |
| **백엔드** | FastAPI (Python) | 비동기 처리, AI 라이브러리 연동 용이 |
| **프론트엔드** | Vue.js 3 + TypeScript | TAS에서 익숙한 스택, 빠른 개발 가능 |
| **스타일링** | Tailwind CSS | 빠른 UI 구성, 모던한 디자인 |
| **컨테이너** | Docker / Docker Compose | 환경 일관성 보장, 배포 편의성 |
| **법령 데이터** | 국가법령정보센터 Open API | 무료, 공식 법령 원문 제공 |

### 4.2 시스템 아키텍처 개요

```
[사용자]
   │ 자연어 질문
   ▼
[Vue.js Frontend]
   │ REST API 호출
   ▼
[FastAPI Backend]
   │
   ├─ 질문 임베딩 (ko-sroberta)
   │
   ├─ 벡터 검색 (ChromaDB)
   │    └─ 유사 조문 Top-K 반환
   │
   ├─ 프롬프트 구성 (LangChain)
   │
   └─ LLM 답변 생성 (GPT-4o-mini)
        │
        ▼
   답변 + 출처(법령명·조문번호) 반환
```
