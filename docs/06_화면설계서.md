## 2. 공통 — 네비게이션 바

### 2.1 화면 구성 요소

**비회원 상태**

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 로고 (⚖️ 법령 RAG) | 텍스트·링크 | 서비스명 | 클릭 시 / 이동 |
| 로그인 버튼 | button | 로그인 페이지 이동 | 클릭 시 /login 이동 |
| 회원가입 버튼 | button | 회원가입 페이지 이동 | 클릭 시 /login?tab=signup 이동 |

**회원 상태**

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 로고 (⚖️ 법령 RAG) | 텍스트·링크 | 서비스명 | 클릭 시 / 이동 |
| 히스토리 | 링크 | 히스토리 조회 페이지 이동 | 클릭 시 /history 이동 |
| 사용자명 표시 | text | "안녕하세요 {username}님" | - |
| 로그아웃 버튼 | button | 로그아웃 처리 | 클릭 시 토큰 삭제 후 / 이동 |

### 2.2 비회원/회원 접근 제어

| 조건 | 처리 방식 |
|------|----------|
| `isAuth === false` | 로그인·회원가입 버튼 표시 |
| `isAuth === true` | 히스토리·사용자명·로그아웃 버튼 표시 |

### 2.3 화면 전환 흐름

```
로고 클릭           → /
로그인 버튼 클릭     → /login
회원가입 버튼 클릭   → /login?tab=signup
히스토리 클릭       → /history
로그아웃 클릭       → 토큰 삭제 → /
```

## 3. UI-01 — 로그인 / 회원가입

### 3.1 화면 구성 요소

**공통**

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 탭 (로그인 / 회원가입) | tab | 로그인·회원가입 전환 | 클릭 시 해당 폼으로 전환 |
| 에러 메시지 영역 | text | API 에러 메시지 표시 | 에러 발생 시 노출, 입력 시 초기화 |

**로그인 탭**

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 이메일 입력 | input[email] | 이메일 입력 | - |
| 비밀번호 입력 | input[password] | 비밀번호 입력 | - |
| 비밀번호 보기/숨기기 | icon button | 비밀번호 표시 전환 | 클릭 시 input type 전환 |
| 로그인 버튼 | button | 로그인 요청 | 클릭 시 POST /auth/login 호출 |

**회원가입 탭**

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 사용자명 입력 | input[text] | 사용자명 입력 | - |
| 이메일 입력 | input[email] | 이메일 입력 | - |
| 비밀번호 입력 | input[password] | 비밀번호 입력 | - |
| 비밀번호 확인 입력 | input[password] | 비밀번호 재입력 | - |
| 회원가입 버튼 | button | 회원가입 요청 | 클릭 시 POST /auth/signup 호출 |

### 3.2 유효성 검사 규칙

**로그인**

| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| 이메일 | 필수, 이메일 형식 | 올바른 이메일 형식을 입력해주세요 |
| 비밀번호 | 필수, 1자 이상 | 비밀번호를 입력해주세요 |

**회원가입**

| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| 사용자명 | 필수, 2~50자 | 사용자명은 2자 이상 입력해주세요 |
| 이메일 | 필수, 이메일 형식 | 올바른 이메일 형식을 입력해주세요 |
| 비밀번호 | 필수, 8자 이상 | 비밀번호는 8자 이상이어야 합니다 |
| 비밀번호 확인 | 비밀번호와 일치 | 비밀번호가 일치하지 않습니다 |

### 3.3 비회원/회원 접근 제어

| 접근 유형 | 처리 방식 |
|----------|----------|
| 비회원 접근 | 정상 표시 |
| 회원이 /login 접근 | /로 리다이렉트 |

### 3.4 화면 전환 흐름

```
로그인 성공   → JWT 저장 → / 이동
로그인 실패   → 에러 메시지 표시 (화면 유지)
회원가입 성공 → 로그인 탭으로 전환 + 성공 메시지 표시
회원가입 실패 → 에러 메시지 표시 (화면 유지)
```

## 4. UI-02 — 메인 검색 화면

### 4.1 화면 구성 요소

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 서비스 소개 문구 | text | "자연어로 묻고, 법령에서 찾고" | - |
| 법령 필터 | select (다중 선택) | 검색 대상 법령 선택 | 미선택 시 전체 법령 검색 |
| 질문 입력창 | textarea | 자연어 질문 입력 | 최대 500자, Enter+Shift 줄바꿈 |
| 글자 수 표시 | text | 현재 입력 글자 수 / 500 | 실시간 업데이트 |
| 검색 버튼 | button | 질문 제출 | 클릭 시 POST /qa/ask 호출 |
| 로딩 스피너 | spinner | API 응답 대기 중 표시 | 응답 도착 시 사라짐 |
| 답변 영역 | div | LLM 생성 답변 표시 | 초기 숨김, 응답 후 노출 |
| 출처 카드 목록 | card | 근거 조문 카드 목록 | 클릭 시 UI-04 모달 열림 |
| 면책 문구 | text | 법적 효력 없음 안내 | 답변 하단 항상 표시 |
| 로그인 유도 문구 | text | 비회원에게만 표시 | "로그인하면 히스토리가 저장됩니다" |
| 예시 질문 버튼 | button | 질문 예시 3개 제공 | 클릭 시 질문 입력창에 자동 입력 |

### 4.2 유효성 검사 규칙

| 필드 | 규칙 | 에러 메시지 |
|------|------|------------|
| 질문 입력창 | 필수, 1자 이상 | 질문을 입력해주세요 |
| 질문 입력창 | 500자 이하 | 질문은 500자 이내로 입력해주세요 |

### 4.3 비회원/회원 접근 제어

| 접근 유형 | 처리 방식 |
|----------|----------|
| 비회원 접근 | 정상 표시, 로그인 유도 문구 노출 |
| 회원 접근 | 정상 표시, 로그인 유도 문구 숨김 |
| 질문 제출 (비회원) | 답변 표시, 히스토리 미저장 |
| 질문 제출 (회원) | 답변 표시, PostgreSQL 히스토리 저장 |

### 4.4 화면 전환 흐름

```
검색 버튼 클릭     → 로딩 스피너 표시 → 답변·출처 카드 노출
출처 카드 클릭     → UI-04 답변 상세 모달 열림
로그인 유도 클릭   → /login 이동
```

## 5. UI-03 — 히스토리 조회

### 5.1 화면 구성 요소

| 요소 | 유형 | 설명 | 동작 |
|------|------|------|------|
| 페이지 제목 | text | "내 질문 히스토리" | - |
| 히스토리 카드 목록 | card | 질문·답변 요약 카드 최신순 표시 | 클릭 시 답변 전체 펼쳐짐 |
| 질문 텍스트 | text | 저장된 질문 내용 | - |
| 답변 요약 | text | 답변 앞 100자 표시 | - |
| 질문 일시 | text | 질문한 날짜·시간 표시 | - |
| 더보기 버튼 | button | 답변 전체 펼치기 | 클릭 시 전체 답변 노출 |
| 페이지네이션 | pagination | 10개씩 페이지 구분 | 페이지 클릭 시 GET /qa/history?page=N 호출 |
| 히스토리 없음 안내 | text | 히스토리가 없을 때 표시 | "아직 질문 내역이 없습니다" |

### 5.2 유효성 검사 규칙

> 히스토리 조회 화면은 입력 필드가 없으므로 유효성 검사 규칙 없음

### 5.3 비회원/회원 접근 제어

| 접근 유형 | 처리 방식 |
|----------|----------|
| 비회원이 /history 접근 | /login으로 리다이렉트 |
| 회원 접근 | 정상 표시 |

### 5.4 화면 전환 흐름

```
비회원 /history 접근  → /login 리다이렉트
히스토리 카드 클릭    → 답변 전체 펼쳐짐 (인라인 확장)
페이지네이션 클릭     → 해당 페이지 히스토리 로드
```

## 8. 화면 전환 흐름 전체 요약

```
[비회원]
접속
  │
  ▼
UI-02 메인 검색 화면
  │
  ├─ 질문 입력·검색 → 답변·출처 표시
  │       └─ 출처 카드 클릭 → UI-04 모달
  │
  ├─ 로그인 버튼 클릭 → UI-01 로그인
  │       └─ 로그인 성공 → UI-02 (회원 상태)
  │
  └─ 회원가입 버튼 클릭 → UI-01 회원가입 탭
          └─ 회원가입 성공 → UI-01 로그인 탭

[회원]
UI-02 메인 검색 화면
  │
  ├─ 질문 입력·검색 → 답변·출처 표시 + 히스토리 저장
  │       └─ 출처 카드 클릭 → UI-04 모달
  │
  ├─ 히스토리 클릭 → UI-03 히스토리 조회
  │
  └─ 로그아웃 클릭 → 토큰 삭제 → UI-02 (비회원 상태)

[공통]
정의되지 않은 URL → UI-05 404 페이지 → / 이동
```
