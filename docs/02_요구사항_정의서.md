# 📋 요구사항 정의서

> **프로젝트명:** 법령 RAG 문서 검색 시스템 (Legal RAG Document Search System)
> **작성일:** 2026년 6월
> **작성자:** 전보경
> **버전:** v1.0

---

## 1. 개요

본 문서는 법령 RAG 문서 검색 시스템의 기능적·비기능적 요구사항을 정의한다.
국가법령정보센터 Open API를 통해 수집한 노동 관련 법령 데이터를 기반으로,
사용자의 자연어 질문에 대해 관련 조문을 검색하고 LLM이 답변을 생성하는 시스템이다.

---

## 2. 사용자 정의

| 사용자 유형 | 설명 |
|------------|------|
| 비회원 | 로그인 없이 자연어 질문·답변 조회 가능, 히스토리 저장 불가 (세션 내에서만 표시) |
| 회원 | 로그인 후 질의응답 히스토리 DB 저장·조회 가능 |
| 관리자 | 법령 데이터 적재 및 시스템 관리 (CLI 또는 관리 API로 처리) |

---

## 3. 기능 요구사항

### 3.1 법령 데이터 적재 (관리자)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-01 | 국가법령정보센터 Open API를 통해 지정된 법령 원문을 수집할 수 있어야 한다 | 필수 |
| F-02 | 수집한 법령 원문을 조문(Article) 단위로 파싱하여 청크로 분할할 수 있어야 한다 | 필수 |
| F-03 | 각 청크에 법령명·조문번호·항번호 메타데이터를 태깅할 수 있어야 한다 | 필수 |
| F-04 | ko-sroberta-multitask 모델로 청크를 임베딩하여 ChromaDB에 저장할 수 있어야 한다 | 필수 |
| F-05 | 이미 적재된 법령은 중복 적재되지 않아야 한다 | 필수 |
| F-06 | 적재 대상 법령은 근로기준법·최저임금법·퇴직급여법·남녀고용평등법·고용보험법 5개로 한정한다 | 필수 |

### 3.2 회원 기능

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-07 | 사용자는 이메일·비밀번호로 회원가입할 수 있어야 한다 | 필수 |
| F-08 | 회원은 이메일·비밀번호로 로그인하여 JWT 액세스 토큰을 발급받을 수 있어야 한다 | 필수 |
| F-09 | 비회원은 로그인 없이 질문·답변 기능을 사용할 수 있어야 한다 | 필수 |
| F-10 | 회원과 비회원의 접근 가능 기능이 명확히 구분되어야 한다 | 필수 |

### 3.3 법령 검색 및 질의응답

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-11 | 사용자(비회원 포함)는 자연어로 질문을 입력할 수 있어야 한다 | 필수 |
| F-12 | 입력된 질문을 임베딩하여 ChromaDB에서 유사도 기반으로 관련 조문 Top-K를 검색할 수 있어야 한다 | 필수 |
| F-13 | 검색된 조문을 컨텍스트로 활용하여 GPT-4o-mini가 답변을 생성할 수 있어야 한다 | 필수 |
| F-14 | 답변과 함께 근거가 된 법령명·조문번호·원문 내용을 출처로 함께 반환해야 한다 | 필수 |
| F-15 | 검색 대상 법령을 필터링할 수 있어야 한다 (전체 또는 특정 법령 선택) | 선택 |
| F-16 | 연관 질문 3개를 자동으로 제안할 수 있어야 한다 | 선택 |

### 3.4 질의응답 히스토리

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-17 | 비회원의 질문·답변은 세션 내에서만 표시되며 DB에 저장되지 않는다 | 필수 |
| F-18 | 회원의 질문·답변·출처 이력은 DB에 저장되어야 한다 | 필수 |
| F-19 | 회원은 이전 질의응답 히스토리를 최신순으로 조회할 수 있어야 한다 | 필수 |

### 3.5 면책 및 안내

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-20 | 모든 답변에 "본 서비스는 법적 효력이 있는 해석을 제공하지 않습니다" 면책 문구를 표시해야 한다 | 필수 |
| F-21 | 법령 데이터 기준일(수집 날짜)을 화면에 표시해야 한다 | 선택 |

## 4. 비기능 요구사항

### 4.1 성능

| ID | 요구사항 |
|----|---------|
| NF-01 | 질문 입력 후 답변 생성까지 응답 시간은 10초 이내여야 한다 |
| NF-02 | 법령 데이터 최초 적재 시간은 30분 이내여야 한다 |

### 4.2 정확성

| ID | 요구사항 |
|----|---------|
| NF-03 | 검색된 조문은 질문과 의미적으로 관련된 내용이어야 한다 |
| NF-04 | 문서에 없는 내용에 대해서는 "해당 정보를 찾을 수 없습니다"로 명확히 안내해야 한다 |

### 4.3 사용성

| ID | 요구사항 |
|----|---------|
| NF-05 | 법률 비전문가도 직관적으로 사용할 수 있는 UI를 제공해야 한다 |
| NF-06 | 모바일 환경에서도 사용 가능한 반응형 레이아웃을 제공해야 한다 |

### 4.4 유지보수성

| ID | 요구사항 |
|----|---------|
| NF-07 | Docker Compose로 전체 환경을 단일 명령으로 실행할 수 있어야 한다 |
| NF-08 | 법령 추가 시 적재 스크립트 실행만으로 확장 가능해야 한다 |

## 5. 제약 사항

| 항목 | 내용 |
|------|------|
| 법령 범위 | 노동 관련 법령 5개로 한정 |
| 법적 효력 | 본 시스템의 답변은 법적 효력이 없으며 참고용으로만 사용 |
| 소셜 로그인 | 소셜 로그인 미구현 (이메일·비밀번호 방식만 지원) |
| 실시간 갱신 | 법령 개정 자동 반영 미지원 (수동 재적재 필요) |
| LLM 비용 | GPT-4o-mini API 사용으로 비용 최소화 |
| 임베딩 | ko-sroberta-multitask 로컬 실행으로 임베딩 비용 없음 |

## 7. 외부 시스템 연동

| 시스템 | 용도 | 비고 |
|--------|------|------|
| 국가법령정보센터 Open API | 법령 원문 수집 | 무료, API 키 필요 |
| OpenAI API (GPT-4o-mini) | 답변 생성 | 유료 (API 키 필요) |
| ChromaDB | 벡터 저장 및 검색 | 로컬 실행 |
| PostgreSQL | 질의응답 히스토리 저장 | Docker 컨테이너 |
